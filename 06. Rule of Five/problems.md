# Задачи

## Задача 00

Да се довърши имплементацията на класа `String` от миналия път.
Да се добавят подходящи методи, реализиращи семантики за преместване.

## Задача 01

Нека е дадена следната дефиниция на клас:

```cpp
class A {
public:
  A() { std::cout << "A()\n"; }
  A(int) { std::cout << "A(int)\n"; }
  A(A const &) { std::cout << "A(A const &)\n"; }
  A(A &&) noexcept { std::cout << "A(A &&)\n"; }
  A &operator=(A const &) { std::cout << "A &operator=(A const &)\n"; return *this; }
  A &operator=(A &&) noexcept { std::cout << "A &operator=(A &&)\n"; return *this; }
  ~A() { std::cout << "~A()\n"; }
};

A mkA() {
  return A();
}

A mkA2() {
  A a;
  return a;
}

A mkA3() {
  A a(4);
  return a;
}

A mkA4(int x) {
  A a;
  if (x < 120)
    return а;
  else
    return A(15);
}

A mkA5() {
  A a;
  return std::move(a);
}
```

Какво ще е поведението на всеки от редовете в следните фрагменти?
Кои методи от голямата шестица ще бъдат извикани?
Какво прави `std::move`?

```cpp
  {
1.  A a1();
2.  A a2 = A();
3.  A a3(A());
4.  A a4 = mkA();
5.  A a5 = mkA2();
6.  A a6 = mkA3();
7.  A a9 = mkA4(121);
8.  A const a7 = a4;
9.  A a8 = std::move(a5);
10. a5 = a6;
11. a6 = std::move(a7);
12. a6 = std::move(a5);
13. A a9 = mkA5();
  }
```

## Задача 02

Да се реализира клас `Quiz`, който представя съвкупност от въпроси и отговори.
Всеки въпрос съдържа низовото представяне на въпроса (низ с произволна дължина)
и четири възможни отговора на този въпрос, като точно един от тях е верен.
Да се реализират начини за добавяне на въпрос към куиза и форматирано
извеждане на въпрос на стандартния изход по подходящ начин.

Реализирайте горния клас, като използвате композиция. За низовото представяне
на въпроса използвайте класа `String` от миналия път, а за низовото представяне
на всеки от отговорите - низ с фиксирана дължина (константа по време на компилация по ваш избор).
Няма ограничение спрямо броя на въпросите, които даден куиз може да съдържа.

## Задача 03

Да се реализира шаблон на клас `unique_ptr`, представящ "уникален" указател към динамично заделен ресурс
от някакъв тип `T`, който не може да бъде копиран и се грижи сам за паметта си.
Класът да поддържа следния интерфейс:

- реализирани методи от голямата шестица по подходящ за условието начин;

- `T &getValue()`, който позволява промяната на данните, които се сочат от указателя;

- `bool isNull() const`, който проверява дали указателят е `nullptr`;

- `T const &getValue() const`, който позволява четене на данните, които се сочат от указателя;

- `T *getPtr()`, аналогично на горните методи;

- `T const *getPtr() const`, аналогично на горните методи;

- `T *release()` - отдава притежание върху обекта, който се сочи от вътрешния указател.

> :warning:
> По принцип в стандартната имплементация се използва предефиниране на операторите `*` за `getValue`
  и `->` за `getPtr`, но тъй като все още не сме говорили за това, ще използваме горните имена на методи.
