# Голяма шестица (Rule of Five)

Предстои да разширим методите от голямата четворка, с които се запознахме миналия път.
Понякога в C++ бихме искали да прехвърлим данните от някакъв обект директно в друг такъв,
понеже повече не бихме се интересували от този обект, а копирането би било прекалено скъпо.
За целта се използват т.нар. семантики за преместване (move semantics), които ефективно ни
позволяват да "откраднем" данните от някакъв обект, който няма да използваме повече.
Семантиките за преместване се реализират чрез два метода на класа -
конструктор за преместване и преместващ оператор за присвояване.

Така методите от голямата шестица се състоят от:

- конструктор по подразбиране;
- копиращ конструктор;
- преместващ конструктор;
- копиращ оператор за присвояване;
- преместващ оператор за присвояване;
- деструктор.

## Категории на стойностите (value categories)

Всеки израз в C++ се характеризира с **тип** и с **категория на стойностите**.
Приемаме, че знаем какво е тип на даден израз и какъв е смисълът от типовете.
Ще разгледаме какво са категории на стойностите и за какво са полезни те.

Всеки израз принадлежи на една от следните три категории на стойностите:
`lvalue`, `xvalue` или `prvalue`.

```
                         expression
                             |
              +--------------+--------------+
              |                             |
        glvalue (has identity)       rvalue (temporary)
              |                             |
      +-------+-------+             +-------+-------+
      |               |             |               |
 lvalue           xvalue         prvalue         xvalue
(named)     (expiring object)   (pure value) (expiring object)
```

- `lvalue` стойностите **винаги** заемат място в паметта. Това са стойностите, които **имат имена (идентичност)** -
  т.е. са дефинирани като променливи (`int x;`), константи (`int const y = 0;` и др.) или функции;

- `xvalue` (`expiring value`) стойностите са *вече изчезващи* обекти, които вече **имат идентичност**,
  но данните от тях могат да бъдат безопасно преместени (напр. резултатът от `std::move(object)` за някакъв обект `object`);

- `prvalue` (`pure value`) стойностите са временни обекти или литерали, които **нямат идентичност** и могат да се използват
  за инициализация или изчисления.

### `std::move`

`std::move` се използва за преместването на данните от даден обект някъде другаде (напр. в друг обект).
Сама по себе си тази функция **не премества нищо** - това, което прави, е, че преобразува даден обект
до rvalue стойност (и по-конкретно - xvalue), като в имплементацията се използва `static_cast` (но не се състои само от това!).

> :warning:
> Това, че в имплементацията на `std::move` се използва `static_cast` и че първата е просто преобразуване до rvalue,
  **не означава, че двете функции са взаимно заменяеми**. В `std::move` се правят допълнителни неща, които гарантират
  коректната работа на функцията.

## Конструктор за преместване (move constructor)

Конструкторът за преместване се използва за конструиране на **нов обект**
чрез прехвърлянето на данните от друг обект в този, който конструираме.
За целта конструкторът за преместване приема rvalue референция към обект от същия тип:

```cpp
class A {
public:
  A(A &&other) noexcept {
    // ...
  }
};
```

## Преместващ оператор за присвояване (move assignment operator)

Преместващият оператор за присвояване се използва за прехвърлянето на данните
от друг обект във вече съществуващ такъв. За целта десният операнд на оператора
за присвояване е от тип rvalue референция към обект от същия тип:

```cpp
class A {
public:
  A &operator=(A &&other) noexcept {
    // ...
    return *this;
  }
};
```

## Пример за голяма шестица в клас за вектор с елементи от произволен тип

```cpp
template<typename T>
class Vector {
public:
  Vector() : m_arr(nullptr), m_size(0), m_capacity(0) {}

  Vector(Vector const &other) {
    copyFrom(other);
  }

  Vector(Vector &&other) noexcept {
    moveFrom(std::move(other));
  }

  Vector &operator=(Vector const &other) {
    if (this != &other) {
      free();
      copyFrom(other);
    }
    return *this;
  }

  Vector &operator=(Vector &&other) noexcept {
    if (this != &other) {
      free();
      moveFrom(std::move(other));
    }
    return *this;
  }

  ~Vector() { free(); }

private:
  void copyFrom(Vector const &other) {
    m_arr = new T[other.m_capacity];
    for (size_t i = 0; i < other.m_size; ++i) {
      m_arr[i] = other.m_arr[i];
    }

    m_size = other.m_size;
    m_capacity = other.m_capacity;
  }

  void moveFrom(Vector &&other) noexcept {
    m_arr = other.m_arr;
    m_size = other.m_size;
    m_capacity = other.m_capacity;

    other.m_arr = nullptr;
    other.m_size = 0;
    other.m_capacity = 0;
  }

  void free() { delete[] m_arr; }

private:
  T *m_arr;
  size_t m_size;
  size_t m_capacity;
};
```

## Избягване на копиране (copy elision)

Избягването на копиране е оптимизация на компилатора, задължителна в дадени ситуации със C++17,
при която не се създават временни обекти:

```cpp
A a = A();
```

Ако следваме правилата за извикване на конструктори, които знаем, то горното твърдение би извикало
първо конструктора по подразбиране за обекта `A()` от дясната страна на `=`, след което би се
извиквал конструкторът за преместване, ако има такъв, или копиращият конструктор (също ако има такъв),
като този за преместване би бил с по-голям приоритет при наличие и на двата вида конструктори.
Но от C++17 в тази ситуация гарантирано ще се извика само един конструктор - този по подразбиране.

### Return-value optimisation

Return-value optimisation (RVO) е частен случай на copy elision, при който не се правят допълнителни копия
на обекти, които се връщат като резултат от функция. Има два вида:

- unnamed return-value optimisation (URVO) - обектът, който се връща като резултат от функция, няма идентичност
  (т.е. не е запазен в локална променлива/константа):

  ```cpp
  A mkA() {
    return A();
  }

  int main() {
    A a = mkA();
  }
  ```

  В горния пример се извиква конструкторът по подразбиране точно веднъж.

  С въвеждането на C++17 като стандарт URVO е винаги гарантирана оптимизация (prvalue semantics),
  т.е. приложима е и в по-сложни примери:

  ```cpp
  A mkA(int x) {
    if (x < 7)
      return A();  // A()
    else
      return A(83); // A(int)
  }

  int main() {
    A a = mkA(19);  // A(int) точно веднъж
    A a2 = mkA(3);  // A() точно веднъж
  }
  ```

- named return-value optimisation (NRVO) - обектът, който се връща като резултат от функция, има идентичност:

  Този вид оптимизация е **невинаги приложим**, особено при наличие на по-сложен контрол на потока.

  Тук NRVO е приложима оптимизация:

  ```cpp
  A mkA() {
    A a;
    return a;
  }

  int main() {
    A a = mkA();  // A() точно веднъж
  }
  ```

  Но не и в тази ситуация:

  ```cpp
  A mkA(int x) {
    A a, a2(10);
    if (x < 93)
      return a;
    else
      return a2;
  }

  int main() {
    A a = mkA(67);   // A() -> A(A &&) или A() -> A(A const &), ако няма move конструктор
    А а2 = mkA(100); // A(int) -> A(A &&) или A(int) -> A(A const &)
  }
  ```
