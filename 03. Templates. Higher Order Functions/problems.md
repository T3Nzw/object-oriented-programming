# Задачи

## Задача 00

Да се дефинират шаблони на следните две функции:

- функция, която приема два елемента от произволен тип и връща по-големия от тях;
- функция, която приема масив с елементи от произволен тип и неговия размер и извежда всичките му елементи в някакъв изходен поток;
- функция, които заделят динамична памет за масив с елементи от произволен тип и го запълва с някаква конкретна стойност.

## Задача 01

Да се дефинира шаблон на функция `equal`, която приема
два параметъра от произволен тип и връща дали те са равни.
Ако е възможно, функцията да се специализира за типове на стойности
с плаваща запетая и за низове.

## Задача 02

Да се дефинира функция, която приема два *масива* и разменя техните елементи.
Какви ограничения имаме върху тези масиви?

## Задача 03

Да се дефинира функция, която приема две функции и връща тяхната композиция.

**Пример**:

```cpp
int plusOne(int x) { return x + 1; }
bool odd(int x) { return x % 2; }

compose(odd, plusOne)(2);  // ~> true
```

## Задача 04

Да се дефинира шаблон на функция от по-висок ред `findBy`, която приема едноместен предикат
и някакъв масив. Функцията да връща информация за стойността и индекса на първия елемент
в масива, който удовлетворява предиката. Ако такъв елемент не съществува, функцията да връща
някакъв елемент от конкретния тип и индекс `-1`.

**Пример:**

```cpp
bool even(int x) { return x % 2 == 0; }
int arr[4]{1,2,3,4};
findBy(even, arr, 4);  // ~> стойност: 2, индекс: 1
```

### Задача 05

Да се дефинират следните три функции от по-висок ред, работещи над масиви:

- `map`, която приема едноместна функция `f:T -> T` и я прилага над всички елементи в масива;
- `filter`, която приема едноместен предикат `p:T -> Bool` и връща масив от всички елементи в масива, които удовлетворяват предиката.

### Задача 06

Да се дефинира тип данни `Down`, който съдържа единствен елемент от произволен тип.
Да се дефинират функции за сравнение на елементи от тип `Down<T>` за някакъв тип `T`,
за който приемаме, че са дефинирани операторите за сравнение (`<`, `<=` и т.н.).
Релацията на линейна наредба над елементите от тип `Down<T>` е обратната релация
на линейна наредба на елементите от тип `T`.

### Задача 07 (*)

Нека си припомним рекурсивната дефиниция на функция, пресмятаща `n`-тото число на Фибоначи:

```cpp
std::size_t fib(std::size_t n) {
  if (n <= 1) return n;
  return fib(n-1) + fib(n-2);
}
```

Ако компилираме и изпълним програма, съдържаща `fib(40)` (приемаме, че оценката на израза се използва),
пресмятането на този израз би отнело известно време. Ако числото, което подадем, времето за изпълнение
също ще се увеличи сравнително. Причината за това е, че `fib(...)` се пресмята по време на изпълнение на
програма.

Да се дефинира начин за пресмятане на числа на Фибоначи, което се извършва по време на *компилация*.

*Жокер*: Да се използват шаблони на структури и енумерации. Защо не бихме могли да използваме шаблони на функции?
